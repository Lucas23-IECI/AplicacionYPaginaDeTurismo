# Decisiones pendientes

Cosas que faltan por definir antes de avanzar en desarrollo:

## Modelo de suscripcion

- Mensual por estar en la plataforma?
- Por cantidad de eventos activos?
- Planes segun visibilidad?
- Hibrido entre suscripcion + destacados?
- Recomendacion inicial: suscripcion mensual + limite de eventos activos + opcion de destacado adicional

## Funcionalidad

- Como se define un evento destacado
- El anunciante puede editar un evento ya enviado o debe reenviarlo
- El administrador puede reescribir completamente el anuncio
- Categorias fijas o etiquetas libres
- El mapa muestra solo eventos futuros
- Multiidioma desde el inicio o despues
- Ficha de organizador ademas de ficha de evento
- El usuario puede guardar favoritos o solo navegar
- Metricas del anunciante: basicas o avanzadas

## Stack final

- Supabase (camino nuevo) vs Prisma + SQL alineado a PuroCode (camino reutilizable)
- Almacenamiento multimedia
- Roles y permisos
- Pasarela de pagos: Stripe vs Transbank

## Presupuesto

- Cerrar alcance funcional antes de presupuestar fino
- No poner rangos de desarrollo sin alcance cerrado
- Separar costos de infraestructura del costo de desarrollo
